Інструкція: GitHub Pages + Telegram Bot

Ця інструкція допоможе створити власний Telegram WebApp, використовуючи GitHub Pages. Всі кроки можна виконувати без серверу — сторінка буде працювати через GitHub Pages.

Крок 1: Підготовка репозиторію на GitHub

1. Увійди у свій акаунт GitHub.

2. Створи новий репозиторій, наприклад: my-webapp.

3. Завантаж туди свої файли: index.html, style.css, script.js.

Якщо файли вже є у репозиторії, переходь до наступного кроку.

Крок 2: Включення GitHub Pages

1. Перейди у вкладку Settings свого репозиторію.

2. Знайди розділ Pages (або прокрути вниз).

3. У полі Source вибери main branch або main branch /root і натисни Save.

4. Після збереження побачиш посилання на GitHub Pages, наприклад: https://username.github.io/my-webapp/

Це посилання буде використовуватися в Telegram Bot для відкриття сторінки.

Крок 3: Створення Telegram Bot

1. Відкрий Telegram і знайди @BotFather.

2. Надішли команду /newbot.

3. Введи назву бота та username (username має закінчуватися на "bot").

4. BotFather надасть токен — збережи його, він потрібен для підключення бота.

Це може бути тестовий бот, він не має залежати від твого основного проекту.

Крок 4: Підключення GitHub Pages до Telegram Bot

1. В коді бота вкажи URL сторінки GitHub Pages як Web App URL або посилання для кнопки.

Приклад на Python (aiogram / pyTelegramBotAPI):

from aiogram import Bot, Dispatcher, types
from aiogram.utils import executor

TOKEN = "YOUR_BOT_TOKEN"
bot = Bot(token=TOKEN)
dp = Dispatcher(bot)

@dp.message_handler(commands=['start'])
async def start(message: types.Message):
    keyboard = types.InlineKeyboardMarkup()
    keyboard.add(types.InlineKeyboardButton("Відкрити WebApp", url="https://username.github.io/my-webapp/"))
    await message.answer("Привіт! Натисни кнопку нижче:", reply_markup=keyboard)

if __name__ == "__main__":
    executor.start_polling(dp)
Для JavaScript бота структура буде аналогічною — важливо вказати URL GitHub Pages.

Крок 5: Тестування

1. Відкрий Telegram і напиши боту /start.

2. Натисни кнопку “Відкрити WebApp”.

3. Повинна відкритись твоя сторінка з GitHub Pages, і всі стилі та скрипти мають працювати.

Поради

  • Всі зміни у файлах index.html, style.css, script.js автоматично оновлюються на GitHub Pages після пушу.
  • Тестуй сторінку в браузері перед підключенням до бота.
  • Якщо бот не відкриває сторінку, перевір URL і права доступу репозиторію (Public).
